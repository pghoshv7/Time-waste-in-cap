Migration Pattern Assessment Questionaries

Is the application a lightweight application with less features and use cases?
Is the application usage very less (number of hits < 100 etc..)?
Is the application have no or minimal backend service dependencies?
Is the application a standalone application with no upstream/downstream dependencies?
Is the application internal application used only by internal employees/staff and not involved in business processes?
Is the application a monolith application with multiple components tightly coupled?
Is application changes frequently due to new business requirements?
Is the application support/maintenance cost is very high?
Is the application's availability a measure concern in existing environment?
Is the application's scalability a measure concern in existing environment?
Is the application resource intensive? (data processing and analysis, report generation etc.. )
Is the application business critical?
Is one more core components of exisitng application has contractual obligation
to continue for certain period of time?
Is the application need to continue using the existing services for various organizational needs?
(Ex: Don't want Data migration, or change security infrastructure etc..)
Is the application need to keep the existing API,
Programming language, Source Code, Business Logic of few of the core components?
