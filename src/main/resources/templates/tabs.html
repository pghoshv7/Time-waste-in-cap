For more information on the migration pattern please click on any of the following tabs:

Re-host

Rehosting: also known as lift-and-shift.
Deploying the application in cloud platform without applying any change in the existing application.
Its very fast and can be done both manually or using automated tools.

Re-platform

Replatforming: This approach will require few configuration changes or a few other cloud optimizations in order to achieve some tangible benefits,
but otherwise not changing the core architecture of the application. Applications which are already hosted in cloud can later be re-desingned easily.

Re-write

Rewrite: This approach requires the existing application to be completely redesigned/re-architectured by applying micro-service based architecture
following best practices of cloud based development. This approach is typically driven by a strong business need to add features, scale, or performance
that would be difficult to achieve in the applicationâ€™s existing environment.
This pattern tends to be the most expensive, but it can also be the most beneficial.

Co-exist

Co-exist: This approach requires moving few of the existing applications to cloud while keeping others in the local data center.
This approach best suited for Companies don't want to deploy all of their applications to cloud because of various reasons(Security, contracts etc..) or those who just started their cloud migration journey.
